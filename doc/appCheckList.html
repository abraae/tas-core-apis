Checklist for building apps

Servers

  • What discrete servers will make up the app?
  • For each server:
    • Is the server stateless? If not, what data does it hold on the file system, or elsewhere?
    • What network traffic goes between this server and other servers making up the app?

Life cycle and provisioning

  • When new tenants arrive:
    • What work is done? e.g inserting into customer tables, provisioning machines, creating S3 folders, etc?
    • How long (in seconds) is new tenant processing expected to take?
    • Does new tenant processing complete immediately, or is there queued work (e.g. replication) before new tenants become fully ready?

Storage

  • What databases, cloud storage (e.g. S3) or other shared storage systems are used by the app?

    • Scalability

    • What is the strategy for horizontal scaling, if required?
    • What new components (load balancers, etc.) are required?
